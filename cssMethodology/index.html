BEM
(Block Element Modifier)

CSS 방법론들 중에서 가장 인기있는 방법론

재사용할 수 있는 컴포넌트를 쉽게 구현할 수 있는
작명 규칙(Naming Convention) 모음

목표는 직관적인 네이밍

naming Convention

							
								.block { ... }
								.block__element { ... }
								.block--modifier { ... }
								.block__element‐‐modifier { ... }

디버깅 시간을 절약 할 수있는 CSS 명명 규칙

block

독립적인 페이지 구성 요소

형태(red, big)가 아닌
목적(menu, button)에 맞게 결정

ex) header, footer, content, navigation, form, menu , search from , 디자인 블록 ...

Element

블록안에서 특정 기능을 담당

형태(red, big)가 아닌
목적(item, text, title)에 맞게 결정

ex) menu__item, header__title …

Modifier

블록이나 요소의 모양(color, size..),
상태(disabled, checked..)를 정의한다.

ex) form__button — disabled ...

OOCSS
(Object Oriented CSS)


원칙1 : 구조(structure)와 모양(skin)의 분리

원칙2 : 콘테이너와 콘텐츠의 분리

목표는 코드의 재사용으로 중복 최소화

naming Convention

  • 가능한 짧고 간결하게 작성한다.
  • 동작과 형태가 예상 가능하도록 명확하게 작성한다.
  • 어떻게 생겼는지 보다는 어떤 목적인지 알 수 있도록 의미있게 작성한다.
  • 지나치게 구체적 이지 않게 일반적으로 사용가능 하도록 작성한다.

원칙1 :
구조(structure)와 모양(skin)의 분리


structure

ex) width, height, margin, padding, overflow ....

skin

ex) color, font, shadows, ....

							
								.btn {
									width: 100px;
									padding: 10px;
								}									
								.btn-danger {
									color: white;

SMACSS
(Scalable and Modular Architecture for CSS)


CSS를 위한 확장가능하고 모듈화된 아키텍처

기본(base), 레이아웃(layout), 모듈(module), 상태(state), 테마(theme) 다섯가지의 범주를 제시한다.

목표는 모듈식 구조로 확장성 증대

SMACSS 범주화(categorization)

  • Base : 스타일 초기화 (reset.css)
  • layout : 주요 요소(id), 하위 요소(class)로 구분하고 'layout => l-' 접두사를 사용
  • module : 재사용 가능한 구성요소 모듈간 접두사 뒤에 '--' 사용
  • state : 요소의 상태 변화를 표현하고 접두사 사용 'is-'
  • theme : 사용자가 선택 가능 하도록 스타일을 재선언하여 사용

ITCSS
(Inverted Triangle CSS)

Harry Roberts가 만든 CSS 아키텍처

reach, specificity, explicitness 기준으로 그룹화

sass 같은 전처리기 사용 base

ITCSS Architecture 7Layer & Key Metrics

reach :페이지에 영향을 미치는 범위

specificity :낮은 특이성에서 높은 특이성으로

explicitness :일반화로 시작하여 (구체적이고) 명확하게

  • Settings : (전처리기) Global variables, config switches. (ex: base, font, color ...)
  • Tools : (전처리기) Default mixins and functions. (일반적으로 클래스의 속성을 작성/확장하는데 도움을 주는 전처리 함수)
  • Generic : 실제로 CSS를 생성하는 첫 번째 레이어, 스타일, 상자 크기 정의 등을 재설정, 정규화하는 Ground-zero styles (ex: reset.css, normalize.css)
  • Base(Elements) : Unclassed HTML elements (like H1, A, etc.) (type selectors)
  • Objects (네임스페이스: o-) : Cosmetic-free design patterns. 객체 클래스에 아이템 정렬 같은 디자인 패턴에 사용
  • Components (네임스페이스: c-) : Designed components, chunks of UI (UI 구성 요소에 대한 스타일 규칙). 독립적인 components (ex: button, dialog ...)
  • Trumps : Helpers and overrides. 기존 규칙을 조정하고 재정의하는 유틸리티 및 도우미 클래스 (!important)
